# Project Vision & Philosophy

## What this project is

This project is a **Telegram Mini App incremental game** designed to run primarily inside the Telegram mobile app.

- Mobile-first design is mandatory.
- Desktop support is secondary and optional.

## Target platform

- Telegram WebView
- Mobile-first layout
- Single-page experience
- No complex routing
- No desktop-specific layouts required

## Product philosophy

- Minimalistic UI
- One dominant action (tap)
- Fast feedback
- Clear state
- No unnecessary features

## Technical philosophy

- Build in small vertical slices
- Always start minimal
- No overengineering
- Backend authoritative
- Shared pure logic
- Deterministic state

## UX philosophy

- Thumb-friendly layout
- Big tap area
- Top status bar
- No tiny clickable elements
- Avoid heavy UI libraries

## What this project is NOT

- Not a generic web dashboard
- Not a desktop-first SaaS
- Not a multi-page website
- Not a microservices architecture
